The Problem

Indic languages have been underserved by neural codecs.

Standard audio codecs are trained predominantly on English and Mandarin. Indic languages carry phonemic distinctions — aspiration, retroflexion, murmured voice — that generic models fail to preserve. Lipika is the first codec designed from first principles for the Indic phonological inventory.

Why This Matters

In Hindi alone, the difference between /pal/ (moment), /p²al/ (fruit), and /p²ːal/ (edge of a blade) is phonemically contrastive. A codec that confuses aspiration or retroflexion produces semantically incorrect tokens. Lipika's Script-Family Adapter ensures these contrasts survive compression.

Architecture

Encoder → Residual VQ → Decoder, conditioned on script.

Built on the EnCodec and SoundStream lineage. Novel contributions include the Script-Family Adapter (AdaLN), retroflex latent prior, and EMA dead-code recycling with MPS-compatible buffer updates.

Technical Innovation

Five architectural contributions beyond the state of the art.

01

Script-Family AdaLN Conditioning

Adaptive Layer Normalisation conditioned on Unicode script family. Unlike language-agnostic codecs that learn a single prior for all phonemes, Lipika injects a 64-dimensional script embedding (projected through SiLU-activated layers) as scale and shift parameters into the encoder's LayerNorm. This provides a script-level inductive bias without increasing codebook size or altering the decoder.

02

Retroflex Latent Prior

Retroflex consonants (/ṭ ḍ ṇ ɽ/) are phonemically contrastive in 10 of 12 supported script families. During initialisation, the script embedding's first 8 dimensions are biased by +0.5 for retroflex-bearing scripts. This soft prior encourages the first VQ codebook to allocate dedicated entries for retroflex place of articulation, improving token discriminability in minimal pairs.

03

Semantic Distillation from W2V-BERT 2.0

The first VQ codebook is trained to predict frozen W2V-BERT 2.0 hidden states (layer 6) via a lightweight projection head. This ensures the coarsest codes capture phonetic identity rather than acoustic texture, enabling the tokenizer to serve as a semantic discretisation layer for downstream TTS (VALL-E style). Subsequent codebooks refine residual acoustic detail.

04

EMA Dead-Code Recycling with MPS Compatibility

Codebook collapse is a well-documented failure mode in VQ-VAEs. Lipika tracks cluster sizes via exponential moving averages and reinitialises dead codes (usage below threshold) from live batch vectors. The implementation uses a clone-modify-copy pattern that is compatible with CUDA, MPS, and CPU — a nontrivial engineering detail given MPS's restriction on in-place buffer mutation.

05

Gaussian Jitter for Quantisation Symmetry Breaking

With small batch sizes (especially on CPU/MPS), multiple encoder frames can collapse onto the same codebook entry. Lipika adds zero-mean Gaussian noise (σ=1e-3) to the projected encoder output during training. This breaks the symmetry that causes mode collapse without affecting inference, dramatically improving codebook utilisation at batch sizes as small as 2.

Quick Start

Deploy in minutes.

Terminal
# Installation
pip install torch soundfile librosa numpy tqdm tensorboard

# Smoke test
python tokenizer.py smoke-test

# Train with auto-detected preset
python tokenizer.py train --preset auto --data-dir ./data --epochs 200

# Encode speech to tokens
python tokenizer.py encode speech.wav \
    --checkpoint ./checkpoints/ckpt_step00005000.pt --lang hi --out codes.pt

# Decode tokens to audio
python tokenizer.py decode codes.pt \
    --checkpoint ./checkpoints/ckpt_step00005000.pt --out reconstructed.wav
